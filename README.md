#### 【开发环境与硬件】

1. 目标硬件：ESP32-C3 Zero 开发板（4MB Flash，RISC-V 单核）
2. 开发框架：Arduino 框架，底层基于 ESP-IDF 原生 FreeRTOS 实时操作系统
3. 工程配置：通过 `platformio.ini` 统一管理，使用 SPIFFS 文件系统，支持双分区 OTA
4. 所有功能以 FreeRTOS 独立任务实现，任务间仅通过队列、事件组通信，禁止直接跨模块调用内部函数
5. 所有模块接口命名统一，返回统一错误码；参数、引脚、任务配置全部集中在公共配置文件
6. 工程遵循 PlatformIO 标准目录结构，私有驱动 / 模块按独立库形式存放于 `lib` 目录
7. 读取规则文档.md
8. 新创建代码的时候需要在开头写上简要的说明，版本默认为1.0，后续以0.1来增加，参考现有的代码
9. 文件结构，参考下面的src，需要方便维护，扩展等

src文件结构参照：

```
src/
├── main.cpp                # 程序入口，初始化+创建所有RTOS任务
├── common/                 # 全局通用配置、日志、错误码
│   ├── config.h
│   ├── log.h
│   └── err_code.h
├── bsp/                    # 底层硬件基础封装(GPIO/UART/PWM)
│   ├── bsp_gpio.cpp
│   └── bsp_gpio.h
├── driver/                 # 外设驱动
│   ├── drv_led.cpp
│   └── drv_led.h
├── service/                # 网络/时间服务
│   ├── srv_wifi.cpp
│   ├── srv_wifi.h
│   ├── srv_mqtt.cpp
│   ├── srv_mqtt.h
│   ├── srv_ntp.cpp

└── app_task/               # FreeRTOS独立任务
    ├── task_wifi.cpp

```

8. 不要创建新的 Python 构建脚本，统一使用 `build.py`。成功之后查看build.log或根据输出信息来来进行判断是否成功。

   ```
   python build.py          # 编译+上传（默认）

   python build.py monitor  # 查看串口输出
   ```

行为指南以减少常见的 LLM 编码错误。根据需要与项目特定说明合并。

权衡：这些指南偏向于谨慎而非速度。对于琐碎的任务，使用判断。

1. 编码前思考
   不要假设。不要隐藏困惑。暴露权衡。

实施前：

明确说明你的假设。如有不确定，请提问。
如果存在多种解释，请呈现它们——不要沉默选择。
如果存在更简单的方法，请说明。在适当的时候提出反对意见。
如果某事不清楚，请停止。指出令人困惑的地方。提问。

2. 先求简单
   解决问题的关键是最少量的代码。不要有任何推测。

不要添加超出要求的功能。
不要为一次性代码使用抽象。
不要“灵活性”或“可配置性”，除非有要求。
不要为不可能发生的场景处理错误。
如果你写了 200 行代码，但实际只需要 50 行，那就重写它。
问问自己：“一个高级工程师会认为这太复杂了吗？”如果会，那就简化。

3. 手术性变更
   只触碰必须修改的部分。只清理自己的烂摊子。

当编辑现有代码时：

不要“改进”相邻的代码、注释或格式。
不要重构那些没有问题的东西。
遵循现有的风格，即使你会有不同的做法。
如果你发现无关的废弃代码，请指出来——不要删除它。
当你的更改产生孤儿节点时：

删除你修改导致未使用的导入/变量/函数。
除非被要求，否则不要删除现有的废弃代码。
测试：每行修改都应该直接追溯到用户的请求。

4. 目标驱动执行
   定义成功标准。循环直到验证通过。

将任务转化为可验证的目标：

"添加验证" → "为无效输入编写测试，然后使其通过"
"修复错误" → "编写一个重现它的测试，然后使其通过"
"重构 X" → "确保在重构前后测试都能通过"
对于多步骤任务，陈述一个简要计划：

1. [Step] → verify: [check]
2. [Step] → verify: [check]
3. [Step] → verify: [check]

4. 编译和上传
   使用根目录的build.py来实现编译和上传，然后查看build.log查看编译和上传情况，如果有问题就进行修正再次编译和上传


6. 遵守当前的开发流程和项目文件结构，目前使用platformio来进行开发和编译，你使用build.py脚本来编译和上传。
