# 使用指南

> 面向后续维护者 / AI：本文档说明本项目结构、编译烧录方式，以及如何在不修改核心文件的前提下扩展功能。

## 1. 项目结构

```
src/
  common/          公共定义（配置、错误码、日志、服务注册）
    config.h
    err_code.h
    log.h
    service_reg.h/.cpp
  bsp/             板级支持
    bsp_debug_usb.h/.cpp   // USB-CDC 串口
    bsp_uart.h/.cpp        // UART1（GPIO20/21）
  service/         服务层
    srv_cli.h/.cpp         // 命令行 + 数字菜单
    srv_sys_info.h/.cpp    // 系统信息菜单
  rtos/            RTOS 任务
    task_def.h
    task_uart.cpp          // 唯一任务：USB 与 UART1 互通 + 菜单
  main.cpp         // 程序入口，仅创建 task_uart
lib/
  Menu/            // 数字菜单库
```

**依赖方向**：`common` → `bsp/service` → `rtos` → `main.cpp`。

## 2. 编译与烧录

项目使用 PlatformIO。根目录提供封装脚本：

```bash
# 编译 + 上传 + 串口监控（默认）
python build.py

# 仅编译
python build.py build

# 仅上传
python build.py upload

# 串口监控
python build.py monitor
```

烧录配置在 `platformio.ini`：
- 环境：`esp32c3`
- 板型：`weactstudio_esp32c3coreboard`
- 框架：Arduino
- 串口：`build.py` 顶部 `UPLOAD_PORT` / `MONITOR_PORT` 变量（默认 COM15）
- 上传波特率：921600，监控波特率：115200

## 3. 扩展方式

新增功能时，优先新建一个 `src/service/srv_xxx.cpp`，通过注册机制接入系统，**不要修改 `task_uart.cpp`、`srv_cli.cpp`、`srv_sys_info.cpp`。**

### 3.1 注册服务

使用 `SERVICE_REGISTER(name, init_fn, priority)` 注册初始化函数，系统启动时按 priority 从小到大调用。

```cpp
#include "common/service_reg.h"

static app_err_t srv_xxx_init(void)
{
    // 初始化资源、注册命令/菜单/信息页
    return APP_OK;
}

SERVICE_REGISTER("xxx", srv_xxx_init, 20);
```

优先级参考：
- `srv_cli`：0
- `srv_sys_info`：5
- 普通业务服务：10 ~ 50
- 依赖网络的服务：60 ~ 100

### 3.2 注册命令

```cpp
#include "service/srv_cli.h"

static void cmd_xxx(const char *args)
{
    (void)args;
    srv_cli_print("xxx ok\r\n");
}

srv_cli_register_cmd("xxx", "xxx 命令", cmd_xxx);
```

命令行模式（非菜单模式）下输入 `xxx` 即可执行。已注册命令会自动出现在 `help` 输出中。

### 3.3 注册菜单

复用 `lib/Menu/Menu.h`：

```cpp
#include <Menu.h>

static void _menu_xxx_action(void)
{
    srv_cli_print("执行 xxx\r\n");
}

static const menu_def_t s_xxx_menu[] = {
    { "执行 xxx", MENU_ACTION, { .action = _menu_xxx_action } },
    { NULL, MENU_ACTION, {0} }
};

menu_register("xxx 工具", s_xxx_menu);
```

### 3.4 注册系统信息页

```cpp
#include "service/srv_sys_info.h"
#include <Menu.h>

static void _show_xxx_info(void)
{
    srv_cli_print("xxx 信息\r\n");
}

srv_sys_info_register_page("xxx 信息", _show_xxx_info);
```

### 3.5 创建任务

使用 `APP_TASK_CREATE` 宏，创建后自动注册到系统信息监控表：

```cpp
#include "rtos/task_def.h"

APP_TASK_CREATE("task_xxx", task_xxx_entry, 4096, 2, NULL);
```

如需检查返回值：

```cpp
app_err_t ret = app_task_create("task_xxx", task_xxx_entry, 4096, 2, NULL);
if (ret != APP_OK) {
    LOG_E("task xxx create failed");
}
```

## 4. 完整示例

`src/service/srv_xxx.cpp`：

```cpp
#include "common/config.h"
#include "common/log.h"
#include "common/service_reg.h"
#include "rtos/task_def.h"
#include "service/srv_cli.h"
#include "service/srv_sys_info.h"
#include <Menu.h>

static void cmd_xxx(const char *args)
{
    (void)args;
    srv_cli_print("xxx cmd\r\n");
}

static void _show_xxx(void)
{
    srv_cli_print("xxx info\r\n");
}

static void _menu_xxx(void)
{
    cmd_xxx(NULL);
}

static const menu_def_t s_xxx_menu[] = {
    { "运行 xxx", MENU_ACTION, { .action = _menu_xxx } },
    { NULL, MENU_ACTION, {0} }
};

static app_err_t srv_xxx_init(void)
{
    srv_cli_register_cmd("xxx", "xxx 命令", cmd_xxx);
    menu_register("xxx 工具", s_xxx_menu);
    srv_sys_info_register_page("xxx 信息", _show_xxx);
    LOG_I("xxx service init");
    return APP_OK;
}

SERVICE_REGISTER("xxx", srv_xxx_init, 20);
```

## 5. 限制与注意事项

- 服务注册表上限：16 个（`service_reg.cpp` 中定义）。
- 命令数量上限：16 个（`srv_cli.cpp` 中 `CLI_MAX_CMDS`）。
- 系统信息页上限：16 个（`srv_sys_info.cpp` 中 `MAX_PAGES`）。
- 任务监控表上限：16 个（`srv_sys_info.cpp` 中 `MAX_TASKS`）。
- 所有注册操作请在服务初始化函数中完成，不要在中断或任务运行时调用。
- 菜单当前为默认交互模式；按 `0` 在根菜单刷新，在子菜单返回上级。
